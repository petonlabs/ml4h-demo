WHAT IS THIS?
─────────────
A real FastAPI that wraps your trained XGBoost readmission model.
Clinicians use APIs (not Jupyter notebooks) to get predictions.


3-STEP STARTUP
──────────────

1. INSTALL (one-time)
   cd week_eight/notebooks
   uv add fastapi "uvicorn[standard]"

2. START SERVER (in terminal 1)
   uv run uvicorn app:app --port 9000

3. TRY IT (in terminal 2)
   uv run python demo_client.py

   OR visit these URLs in your browser:
   • http://localhost:9000          (landing page, explains APIs)
   • http://localhost:9000/docs     (Swagger UI — interactive testing)
   • http://localhost:9000/redoc    (ReDoc — formatted docs)


WHAT CAN YOU DO?
────────────────

Make a single prediction:
  curl -X POST http://localhost:9000/predictions \
    -H "Content-Type: application/json" \
    -d '{"age":"[60-70)","num_medications":16,...}'

Predict 50 patients at once:
  curl -X POST http://localhost:9000/predictions/batch \
    -H "Content-Type: application/json" \
    -d '{"patients":[...]}'

Check if model is ready:
  curl http://localhost:9000/health

See which features matter most:
  curl http://localhost:9000/model/features

Get random test patient:
  curl http://localhost:9000/patients/sample

THE BIG IDEA
────────────
Your model is trained. Now clinicians need to use it.

❌ They can't load your notebook (no Python, no permissions)
❌ They can't SSH into your computer
❌ Predictions must be logged and auditable

✅ An API solves this: clinician sends JSON → server returns prediction

That's deployment. That's this app.


TROUBLESHOOTING
───────────────
"Port 9000 is in use"
→ Use a different port: uv run uvicorn app:app --port 9001

"ModuleNotFoundError: fastapi"
→ Install: uv add fastapi "uvicorn[standard]"

"FileNotFoundError: readmission_pipeline.joblib"
→ Run week8_mon_full_pipeline.ipynb first to train the model

Server won't start?
→ Check: does the model file exist?
   ls ../data/readmission_pipeline.joblib
